AntiBotLiteJS  – защита от накрутки поведенческих метрик

Скрипт для защиты сайта от ботов, которые пытаются накрутить поведенческие метрики (Яндекс.Метрика, Google Analytics и др.).
Анализирует поведение пользователя (движения мыши, клики, нажатия клавиш, прокрутку, сенсорные события) и признаки headless-среды.
При подозрении на бота показывает капчу с перетаскиванием пазла (двухосевая drag-and-drop). После успешного прохождения капчи устанавливается кука и загружается поведенческая метрика.
Особенности

    🧠 Поведенческий анализ – траектория мыши, скорость, линейность, прыжки, длительность кликов, отсутствие движений.

    🖥️ Детекция окружения – navigator.webdriver, плагины, window.chrome, WebGL рендерер.

    🧩 Капча-пазл – пользователь перетаскивает вырезанный фрагмент на выделенную область (дырка). Положение дырки каждый раз случайное.

    🍪 Кука верификации – после успешной проверки ставится кука, чтобы не беспокоить повторно.

    📊 Готовое место для метрики – отдельная функция, куда можно вставить код Яндекс.Метрики, Google Analytics или любой другой системы.

    📱 Поддержка сенсорных устройств – перетаскивание работает и на мобильных.

Установка и подключение

    Скопируйте содержимое файла antibot.js в отдельный файл на вашем сервере.

    Подключите скрипт на нужных страницах перед закрывающим тегом </body>:

html

<script src="/path/to/antibot.js"></script>

Скрипт автоматически запустится после загрузки DOM.
Конфигурация

Все настройки находятся в объекте CONFIG в начале файла. Измените их под свои нужды:
Параметр	Тип	Значение по умолчанию	Описание
OBSERVATION_TIME_MS	число	8000	Время сбора поведенческих данных (мс).
SUSPICIOUS_SCORE_THRESHOLD	число	2	Порог подозрительности. Чем меньше, тем чаще будет показываться капча.
COOKIE_NAME	строка	'human_verified'	Имя куки для хранения факта верификации.
COOKIE_DAYS	число	30	Срок жизни куки (дней).
mouseLinearityThreshold	число	0.98	Линейность траектории мыши выше этого значения считается подозрительной.
jumpThreshold	число	150	Расстояние (px), при котором перемещение мыши считается «прыжком».
minClickDuration	число	30	Минимальная длительность клика (мс). Клики короче считаются подозрительными.
CAPTCHA_WIDTH	число	450	Ширина области капчи (px).
CAPTCHA_HEIGHT	число	250	Высота области капчи (px).
PUZZLE_SIZE	число	70	Размер пазла (квадрат, px).
TOLERANCE	число	12	Допустимое отклонение при совмещении пазла с целью (px).

Как это работает

    Проверка куки – если кука human_verified уже есть, сразу загружается метрика.

    Сбор данных – в течение OBSERVATION_TIME_MS скрипт собирает события мыши, клики, клавиатуру, прокрутку, касания.

    Анализ – вычисляется «балл подозрительности» на основе:

        наличия движений мыши при кликах/клавишах;

        полного отсутствия взаимодействий;

        линейности траектории мыши;

        средней скорости мыши;

        резких прыжков курсора;

        слишком коротких кликов;

        отсутствия touch-событий на сенсорных устройствах;

        признаков headless-среды (navigator.webdriver, плагины, window.chrome, программный WebGL).

    Принятие решения – если балл ≥ SUSPICIOUS_SCORE_THRESHOLD, показывается капча-пазл. Иначе пользователь считается человеком.

    Капча – пользователь перетаскивает фрагмент изображения на выделенную область. Положение цели каждый раз случайное. При успешном совмещении (с учётом TOLERANCE) капча закрывается.

    Верификация – устанавливается кука и вызывается loadMetrics() для загрузки метрики.

Требования

    Современный браузер с поддержкой ES6 (все актуальные версии Chrome, Firefox, Safari, Edge).

    Для корректной работы капчи необходимо, чтобы страница была загружена в обычном (не iframe) окружении.

    Скрипт не требует внешних зависимостей (не использует jQuery, reCAPTCHA и т.д.).
